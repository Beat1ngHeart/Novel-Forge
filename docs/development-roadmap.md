# Novel Forge — 开发路线图

## 阶段一：项目骨架 ✅
- FastAPI + Next.js + PostgreSQL 初始化
- Docker Compose
- 基础 CRUD 和健康检查

## 阶段二：文件上传与章节切分 ✅
- TXT/Markdown 文件上传
- 自动章节识别与切分
- 章节列表和正文查看
- 前端资料库页面

## 阶段三：统一 LLM Provider ✅
- 统一 Provider 接口（chat/structured_output/embedding/count_tokens/health_check）
- Mock Provider（模拟成功/失败/超时/限流）
- Provider Registry（环境变量驱动）
- 调用日志记录（llm_call_logs）
- Provider 管理 API 和前端设置页

## 阶段四：小说项目 CRUD ✅
- 完整项目管理（创建/编辑/归档/恢复/删除）
- 搜索、筛选、排序、分页
- 项目统计
- 前端 4 个页面（列表/新建/详情/设置）

## 阶段五：文本资料上传与权利登记 ✅
- 文件上传（TXT/MD，MIME 校验、SHA-256 去重、安全文件名、防路径穿越）
- 6 种权利状态自动推导分析/参考权限
- 前端 3 个页面（列表/上传/详情）

## 阶段六：文本清洗、卷识别和章节切分 ✅
- 编码检测与清洗（多编码支持、控制字符清理、换行标准化）
- 卷标题识别（第X卷/篇）
- 章节标题识别（中文/英文/数字/特殊章节）
- 事务性解析（幂等、失败不丢数据）
- 人工编辑（改名/排序/合并/拆分/卷/标记）
- 前端 2 个页面（解析预览/章节列表）

## 阶段七：故事基因 Schema 设计 ✅
- 版本化 StoryGene Schema（Pydantic v2，13 个枚举类型）
- 7 大字段组：基础/冲突/情绪/悬念/状态变化/伏笔/文本统计
- 数值范围约束（intensity 0-10, ratio 0-1, confidence 0-1）
- Mock Provider 返回合法数据
- JSON Schema 导出
- 版本注册表（支持后续升级）
- 数据库模型（story_genes 表）
- 29 个测试

## 阶段八：单章故事基因分析闭环 ✅
- 权利检查（prohibited 阻止分析）
- 构建受控 prompt + 调用统一 Provider
- StoryGene Schema 验证 + 失败重试
- chapter_analyses 表存储（不覆盖历史）
- LLM 调用日志
- 前端分析页面（历史/结果/重新分析）
- 19 个测试

## 阶段九：多章节批量分析与任务状态 ✅
- AnalysisTask + AnalysisTaskItem 模型
- 批量任务创建、状态追踪、进度展示
- 单章失败隔离、失败重试、取消
- 并发防护、已分析不重复
- 前端批量分析页面（选择/进度/历史）
- 15 个测试

## 阶段十：小说圣经基础数据管理 ✅
- 4 张表：bible_characters、bible_world_rules、bible_plot_threads、bible_foreshadowings
- 完整 CRUD + 来源状态（ai_suggestion/user_confirmed/text_fact/deprecated）
- 前端小说圣经页面（4 个标签页 + 来源状态标签 + 废弃/恢复）
- 项目隔离 + 级联删除
- 23 个测试

## 阶段十一：原创创意方向生成 ✅
- 3 个结构不同方向（能力/冲突/循环差异）
- 采用/拒绝/编辑/融合
- AI 原始 + 人工修改记录
- 前端创意方向页面
- 13 个测试

## 阶段十二：根据已采用创意生成小说圣经候选 ✅
- BibleCandidate 模型（7 种类别）
- 从已采用方向生成候选
- 逐项确认/拒绝/编辑
- 应用到正式圣经 + 撤销
- 事务保护
- 14 个测试

## 阶段十三：大纲分层生成（下一步）
- 大纲分层生成
- 章节节奏表

## 阶段十三：全书总纲生成与版本管理 ✅
- NovelSynopsis 模型（版本化）
- 14 个字段总纲生成
- 版本管理：draft/adopted/superseded
- 编辑/采用/恢复/比较
- 前端总纲页面
- 14 个测试

## 阶段十四：分卷大纲 ✅
- VolumeOutline 模型（17 字段）
- 从正式总纲生成全部分卷
- 单卷重新生成（不影响其他卷）
- 人工编辑/采用/卷序调整
- 前端分卷大纲页面
- 14 个测试

## 阶段十五：章节节奏表 ✅
- ChapterRhythmPlan 模型（17 字段）
- 批量生成（10 种不同章节功能）
- 单章编辑/重新生成/插入/删除/排序
- AI 原始保留 + 人工修改记录
- 前端节奏表页面
- 14 个测试

## 阶段十六：单章策划器 ✅
- ChapterPlan 模型（17 字段 + 字段锁定）
- 3 个不同方案（行动/调查/内心）
- 采用/拒绝/编辑/重新生成/融合
- AI 原始保留 + 人工修改记录
- 前端单章策划页面
- 13 个测试

## 阶段十七：正文生成器 ✅
- ChapterDraft 模型（20 字段 + 候选数据）
- 从正式策划生成正文草稿
- 文风参数（10 项）
- 人工修订不覆盖 AI 草稿
- 段落级重写
- 候选数据需用户确认
- 前端正文编辑页面
- 11 个测试

## 阶段十八：正文编辑器和版本管理 ✅
- DraftVersion 模型（5 种版本类型）
- 版本保存/恢复/比较/标记正式
- AI 改写/扩写/压缩/语气调整
- 增强编辑器（自动保存/搜索替换/全屏/版本历史）
- 14 个测试

## 阶段十九：正文状态变化提取与人工确认 ✅
- StateChangeCandidate 模型（14 种变化类型）
- AI 提取 + 逐项接受/拒绝/撤销
- 接受自动更新小说圣经
- 事务保护 + 来源记录
- 前端状态变化页面
- 12 个测试

## 阶段二十：一致性审计（下一步）
- 单章策划（3 方案）
- 正文草稿生成
- 一致性审计
- 原创性检查
